# ==========================================
# 📝 Commit Convention (Team Standard)
# ==========================================
# [형식]
# <이모지> <타입>: <간단한 제목> (영문/한글 모두 가능)
#
# [제목 규칙]
# - 50자 이내로 간결하게
# - 첫 글자는 소문자, 마침표 금지
# - 명령형 어조 사용 (ex: "add", "update", "fix")
#
# [본문 규칙]
# - 선택 사항 (있으면 72자 이내 줄바꿈)
# - 무엇을, 왜, 어떻게 변경했는지 설명
# - 관련된 이슈나 참고사항 기록
#
# [푸터 규칙]
# - 이슈 트래킹: Closes #123, Resolves JIRA-456
# - BREAKING CHANGE: 호환성 깨지는 변경 사항 설명
#
# ==========================================
**Commit Title Type**

- ✨ feat : 새로운 기능 추가
- 🐛 fix : 기능 수정, 버그 수정
- 💡 chore : 오타 수정 및 새로운 기능이 추가되지 않고, 코드가 변경 된 경우 (주석 추가 및 수정 포함)
- 📝 docs : 문서 수정 (readme 수정 시)
- 🚚 build : 빌드 관련 파일 수정 및 삭제한 경우
- ✅ test : 테스트 코드, 리팩터링 테스트 코드 추가(프로덕션 코드 변경 X)
- ♻️ refactor : 코드 리팩터링
- 🚑 hotfix : 긴급 수정
# ==========================================#
